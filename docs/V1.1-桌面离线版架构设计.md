# V1.1 桌面离线版架构设计

## 1. 版本定位

V1.1 桌面离线版面向 Windows 优先场景，目标是让合伙人收益系统像普通桌面软件一样安装、启动和离线使用。

- 桌面壳：Tauri
- 前端：沿用 React + TypeScript + Vite
- 本地数据库：SQLite
- 数据保存位置：Tauri 应用数据目录，不放在项目源码目录
- 数据迁移方式：通过完整 JSON 备份导出 / 导入
- 不做云同步
- 不做多用户协作
- 不做自动更新
- 不自动读取浏览器 IndexedDB

## 2. 总体架构

```text
src/domain/
  calculation.ts        核心收益计算逻辑，保持不变
  validation.ts         业务校验，保持现有语义

src/storage/
  repository.ts         存储接口
  indexedDbRepository.ts Web 版 IndexedDB 实现
  sqliteRepository.ts   桌面版 SQLite 实现
  repositoryFactory.ts  根据运行环境选择存储实现
  sqliteSchema.ts       SQLite 表和 schemaVersion 定义

src-tauri/
  tauri.conf.json       Tauri 窗口、构建、插件配置
  src/lib.rs            SQL 插件初始化和迁移注册
  migrations/           SQLite 初始表结构
```

页面层继续通过 `PartnerRepository` 读写数据，不直接访问 IndexedDB 或 SQLite。Web 运行时使用 IndexedDB，Tauri 运行时延迟加载 SQLite 仓库。

## 3. 存储切换规则

- 普通浏览器 / GitHub Pages：`IndexedDbRepository`
- Tauri 桌面运行时：`SqliteRepository`

运行时判断通过 Tauri 注入的全局标记完成。为了避免 Web 包在普通浏览器中提前初始化桌面插件，SQLite 仓库采用动态导入。

## 4. SQLite 表设计

SQLite 当前采用“完整业务对象 JSON payload + 常用索引列”的基础架构，便于 V1.1 快速落地，同时保留后续迁移空间。

核心表：

- `members`
- `capital_lots`
- `capital_transactions`
- `monthly_settlements`
- `monthly_allocations`
- `adjustment_records`
- `dividend_payments`
- `annual_dividend_confirmations`
- `profit_calculator_records`
- `operation_logs`
- `app_metadata`

每条业务记录的金额和比例仍按字符串保存，完整对象保存在 `payload` 字段中。索引列包括 `month`、`year`、`member_id`、`settlement_id`、`status`、`name`、`created_at`、`updated_at`、`record_type`。

## 5. schemaVersion

桌面 SQLite 版新增 `app_metadata` 表，当前：

```text
schemaVersion = 1
```

后续如需要把 JSON payload 拆成更细的关系表，应通过新的 SQLite migration 升级，不直接覆盖旧库。

## 6. 账务保护

桌面 SQLite 版保留 Web 版已有保护：

- locked / adjusted 月份不能直接保存覆盖
- 调整必须通过调整记录
- 导入备份时校验 locked / adjusted 月份收益明细
- 异常 allocation 不直接写入
- 危险全量替换只允许空库演示数据初始化
- 一键清除本地数据仍要求明确确认短语

## 7. 与 Web 版关系

V1.1 桌面版不替代 V1.0.2 Web 版：

- `npm run build` 继续构建 GitHub Pages Web 版
- `npm run desktop:build` 构建桌面前端资源，使用相对路径
- `npm run tauri:dev` 启动桌面开发环境
- `npm run tauri:build` 构建 Windows 安装包

## 8. 当前已知限制

- 第一版不自动读取浏览器 IndexedDB，需要用户手动导出 JSON 再导入桌面版。
- 当前未实现启动密码，仅保留后续规划。
- 当前未实现自动更新。
- 当前未实现 SQLite 数据库副本导出按钮，可在 V1.1.x 补充。
- 本机未安装 Rust 工具链时，无法生成 Tauri 安装包。
