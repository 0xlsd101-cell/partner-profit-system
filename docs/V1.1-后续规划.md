# V1.1 后续规划

## 1. 版本目标

V1.1 建议围绕“离线桌面化、数据安全增强、导出体验增强、维护成本降低”推进，不改变 V1.0 已确认的核心收益业务规则。

## 2. 建议规划项

| 优先级 | 方向 | 说明 |
| --- | --- | --- |
| P1 | Tauri + SQLite 离线桌面版 | 将当前本地网页版迁移为桌面离线软件，使用 SQLite 承载正式数据。 |
| P1 | 启动密码 | 增加本地启动访问保护，降低误打开和误操作风险。 |
| P1 | 本地数据库加密 | 针对 SQLite 数据文件增加本地加密方案评估。 |
| P1 | 自动备份提醒 | 根据上次备份时间、锁定结算和支付动作提醒导出备份。 |
| P2 | 依赖安全专项复查 | 跟踪 `exceljs / uuid` 风险，正式外发前复核依赖链。 |
| P2 | 移动端专项 QA | 对窄屏、横向表格、弹窗、按钮可点击区域做完整走查。 |
| P2 | 源码瘦身专项 | 清理疑似未使用组件、旧模板资产和旧工具函数，单独验证后再删除。 |
| P2 | Excel 导出进一步增强 | 继续优化金额单元格数值化、冻结窗格、筛选、列宽和风险样式。 |
| P2 | PDF 分红确认单 | 增加适合发送和归档的 PDF 确认单。 |
| P2 | 批量导出全部合伙人个人明细 | 支持一次性导出所有合伙人的年度明细或分红条。 |

## 3. 不建议在 V1.1 中改变的内容

- 年化单利收益率口径。
- 折合月收益率 = 年化收益率 ÷ 12。
- 月中加入首月按天折算、次月整月计息。
- 负责人自有资金、负责人专项收益、外部资金差额留存口径。
- 尾差默认归负责人。
- 公历自然年度周期。
- locked / adjusted 历史月份不可直接修改。

## 4. 建议实施顺序

1. 依赖安全专项复查。
2. 移动端专项 QA。
3. 自动备份提醒和启动密码。
4. Tauri + SQLite 技术验证。
5. SQLite 数据加密方案评估。
6. Excel / PDF 导出增强。
7. 批量导出合伙人个人明细。
8. 源码瘦身专项。

## 5. 验收建议

- 所有 V1.0 核心金额测试必须继续通过。
- 不重算 V1.0 locked / adjusted 历史月份。
- 桌面版迁移必须提供可回退备份。
- SQLite 迁移必须有独立迁移测试和恢复演练。
- 新增导出功能不得改变正式账务数据。
- 密码和加密功能不得让用户失去备份恢复能力。
