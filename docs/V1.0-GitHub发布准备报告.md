## 2. GitHub Pages 配置

Vite 已配置自动 base 规则：

- 如果 GitHub Actions 中存在 `GITHUB_REPOSITORY`，普通仓库自动使用 `/<仓库名>/`。
- 如果仓库名以 `.github.io` 结尾，自动使用 `/`。
- 如果本地无法确认仓库名，默认使用 `/`，不猜测仓库名。
- 如需手动覆盖，可设置 `VITE_BASE_PATH`。

GitHub Actions 部署流程：

1. push 到 `main` 分支触发。
2. 支持 `workflow_dispatch` 手动触发。
3. 执行 `npm ci`。
4. 执行 `npm run test`。
5. 执行 `npm run lint`。
6. 执行 `npm run build`。
7. 上传 `dist/`。
8. 使用 `actions/deploy-pages` 部署到 GitHub Pages。

## 7. 仍需用户手动完成的事项

请先安装 Git，或在已有 Git 环境中执行以下命令：

```bash
git init
git branch -M main
git status
```

确认 `git status` 中没有以下内容：

- `release-evidence/`
- `data/trial/`
- `data/archive/`
- `.env`
- 真实 JSON 备份
- 真实 Excel / CSV 报表
- `node_modules/`
- `dist/`

然后提交并打标签：

```bash
git add .
git commit -m "release: v1.0.0 stable web version"
git tag -a v1.0.0 -m "合伙人收益系统 V1.0 网页稳定版"
```

在 GitHub 创建仓库后配置远程：

```bash
git remote add origin https://github.com/<用户名>/<仓库名>.git
git push -u origin main
git push origin v1.0.0
```

GitHub Pages 设置：

1. 进入 GitHub 仓库。
2. 打开 Settings -> Pages。
3. Build and deployment 的 Source 选择 GitHub Actions。
4. 打开 Actions 查看部署。
5. 部署成功后访问对应 Pages 链接。
