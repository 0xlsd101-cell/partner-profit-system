# V1.1 桌面版打包说明

## 1. 前置依赖

桌面版打包需要：

- Node.js
- npm
- Rust 工具链：`cargo`、`rustc`
- Windows WebView2 运行时

本项目已新增 Tauri 依赖：

- `@tauri-apps/api`
- `@tauri-apps/plugin-sql`
- `@tauri-apps/cli`
- Rust 侧 `tauri`
- Rust 侧 `tauri-plugin-sql`，启用 SQLite feature

## 2. 常用命令

Web 版：

```bash
npm.cmd run test
npm.cmd run lint
npm.cmd run build
```

桌面开发：

```bash
npm.cmd run tauri:dev
```

桌面安装包构建：

```bash
npm.cmd run tauri:build
```

如 PowerShell 拦截 `npm.ps1`，继续使用 `npm.cmd`。

## 3. 构建流程

`npm.cmd run tauri:build` 会执行：

1. `npm.cmd run desktop:build`
2. TypeScript 编译
3. Vite 使用相对路径构建桌面前端资源
4. Tauri 编译 Rust 壳
5. 打包 Windows 安装包

Web 版 `npm.cmd run build` 不变，仍用于 GitHub Pages。

## 4. 产物位置

Tauri 构建成功后，Windows 安装包通常位于：

```text
src-tauri/target/release/bundle/
```

具体目录可能包含：

- `nsis/`
- 安装程序 `.exe`

实际路径以构建输出为准。

## 5. 发布包排除项

不得打入或上传：

- `.env`
- 密钥文件
- `release-evidence/`
- `data/trial/`
- `data/archive/`
- 真实 JSON 备份
- 真实 CSV / XLSX 报表
- `node_modules/`
- `dist/`
- `src-tauri/target/`

`.gitignore` 已包含 `src-tauri/target/`。

## 6. 构建失败处理

如果提示缺少 `cargo` 或 `rustc`：

1. 安装 Rust 工具链。
2. 重新打开终端，确认：

```bash
cargo --version
rustc --version
```

3. 再执行：

```bash
npm.cmd run tauri:build
```

不得在缺少 Rust 工具链时声称安装包已生成。

## 7. 当前限制

- 第一版不做代码签名。
- 第一版不做自动更新。
- 第一版不强制启动密码。
- 第一版不自动迁移浏览器 IndexedDB。
- 如 NSIS 构建受本机环境影响失败，可先使用 Tauri 生成的可执行文件做内部验收，再补正式安装器。
