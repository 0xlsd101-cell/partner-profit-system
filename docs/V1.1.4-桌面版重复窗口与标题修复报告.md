# V1.1.4 桌面版重复窗口与标题修复报告

## 1. 问题结论

- 是否复现两个真实窗口：已复现多实例风险。修复前在已打开桌面版的情况下再次点击桌面快捷方式，会新增一个 `partner-profit-system.exe` 进程和第二个桌面窗口。
- 根因：桌面版缺少单实例保护；同时 Tauri 原生窗口标题固定为旧文案“合伙人收益系统 V1.1 桌面离线版”，未随真实版本号更新。

## 3. 根因分析

| 检查项 | 结论 |
| --- | --- |
| Tauri windows 配置 | 仅配置了一个窗口，但未显式设置 `label: main`，标题固定为旧 `V1.1` 文案 |
| Rust 端窗口创建 | 未发现 `WebviewWindowBuilder`、`WindowBuilder` 或额外建窗逻辑 |
| 前端 window.open | 未发现启动时触发新窗口的前端逻辑 |
| 旧快捷方式 / 旧安装目录 | 桌面和开始菜单快捷方式均指向当前安装目录，旧标题来自配置和已安装版本 |
| 多实例情况 | 修复前重复启动会创建第二个进程和窗口 |
