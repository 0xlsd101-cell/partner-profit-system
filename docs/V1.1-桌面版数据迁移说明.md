# V1.1 桌面版数据迁移说明

## 1. 迁移原则

V1.1 桌面离线版不自动读取浏览器 IndexedDB，也不做云端同步。推荐迁移方式是：

1. 在 Web 版导出完整 JSON 备份。
2. 在桌面版导入完整 JSON 备份。
3. 桌面版写入本地 SQLite。
4. 导入前查看中文摘要。
5. 导入前二次确认。

这样可以避免自动读取浏览器数据导致误迁移、误覆盖或跨环境污染。

## 2. 从 Web 版迁移到桌面版

### 2.1 Web 版导出

在 Web 版进入“数据备份”页面：

1. 点击“导出完整备份文件”。
2. 保存 JSON 备份文件到安全位置。
3. 不要手动编辑 JSON 备份文件。
4. 建议同时保留一份外部备份。

### 2.2 桌面版导入

在桌面版进入“数据备份”页面：

1. 选择 Web 版导出的 JSON 备份文件。
2. 查看系统展示的中文数据摘要。
3. 确认导入影响。
4. 二次确认后导入。
5. 导入后核对合伙人、资金批次、月度结算、年度汇总和分红支付记录。

## 3. 导入保护

桌面版导入仍保留以下保护：

- 已锁定 / 已调整月份不允许被静默覆盖。
- 收益明细必须能关联到对应结算月份。
- 无法关联 settlementId 或 month 的异常收益明细会被阻止。
- 导入会记录操作日志。
- 普通用户界面不展示 JSON 原文。

## 4. 数据存储位置

SQLite 文件由 Tauri SQL 插件创建在应用数据目录下，连接名为：

```text
sqlite:partner-profit-system.sqlite
```

该路径相对 Tauri 应用数据目录，不在项目源码目录，也不应被提交到 GitHub。

## 5. 备份建议

- 每次真实月份录入前先导出完整备份。
- 锁定月度结算前先导出完整备份。
- 锁定后再导出一次完整备份。
- 恢复演练应在独立环境中进行。
- 不要把真实 JSON 备份、真实 CSV、真实 XLSX 外发或提交到仓库。

## 6. 不做的事情

V1.1 第一版不做：

- 自动读取浏览器 IndexedDB。
- 自动同步 GitHub Pages 数据。
- 多设备自动同步。
- 云端备份。
- 多用户协作。
- 自动更新。
