# V1.1.1 桌面版清除本地数据修复报告

## 1. 修复结论

- 修复结果：已修复。
- 建议版本：建议发布 v1.1.1。
- 影响范围：仅涉及本地数据清除流程、SQLite repository 写入执行方式和清除确认文案。
- 是否修改收益业务逻辑：否。

## 2. 问题原因

桌面版已经切换到 Tauri + SQLite 存储，但 SQLite repository 的写入流程使用了手写 `BEGIN IMMEDIATE` / `COMMIT` / `ROLLBACK` 包装。

在 Tauri SQL 插件的桌面 WebView 运行环境中，该手写事务包装会导致写入流程被拒绝，表现为页面点击“我已备份，确认清除”后数据没有被清除，或显示清除失败。

同时，原清除逻辑会在清空后再写入一条操作日志，导致“清除后空库”的用户预期与实际摘要不一致。

## 4. Web IndexedDB 清除逻辑

Web 版继续通过 repository 抽象执行清除，不直接在页面层访问 IndexedDB。

清除范围包括：

- 合伙人
- 资金批次
- 资金流水
- 月度结算
- 月度收益明细
- 分红支付记录
- 调整记录
- 年度分红确认单
- 收益计算器记录
- 操作日志

清除后不再写入“本次清除日志”，确保摘要显示为空库。

## 5. Desktop SQLite 清除逻辑

桌面版通过 `SqliteRepository.clearLocalData` 执行 SQLite 清除。

清除方式：

- 不删除 SQLite 文件。
- 不删除 `app_metadata`。
- 保留 `schemaVersion = 1`。
- 清空所有业务表。
- 清除 SQL 由 `sqliteClearBusinessDataSql()` 统一生成。

已清空业务表：

- `members`
- `capital_lots`
- `capital_transactions`
- `monthly_settlements`
- `monthly_allocations`
- `dividend_payments`
- `adjustment_records`
- `annual_dividend_confirmations`
- `operation_logs`
- `profit_calculator_records`

## 6. UI 刷新机制

清除成功后页面继续执行：

- 关闭导入摘要状态。
- 清空确认短语和原因输入。
- 调用 `reload()` 重新读取 repository。
- 显示成功提示：“本地数据已清除。”
- 数据备份页摘要刷新为 0。
- 首页重启后显示空库状态。

失败时显示中文提示：

“本地数据清除失败，请先导出备份后重试。”

## 7. 安全文案

当前确认文案：

- 标题：危险操作：清除本地数据
- 范围说明：此操作将清除当前设备中的本地业务数据，包括合伙人、资金批次、月度结算、收益明细、分红支付、操作日志等记录。
- 恢复说明：此操作不会自动生成备份，清除后无法从当前设备恢复。
- 备份提醒：请先导出完整备份文件，再继续操作。
- 确认短语：确认清除本地数据
- 按钮：我已备份，确认清除

确认短语不正确时，清除按钮保持不可用，不会执行清除。

## 10. 风险与说明

- 本次未修改收益计算、年化收益率、首月折算、负责人收益、外部资金差额留存、尾差归属或年度周期规则。
