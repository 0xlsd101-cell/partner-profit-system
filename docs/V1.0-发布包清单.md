# V1.0 发布包清单

## 1. 建议包含

V1.0 网页稳定版发布包建议包含以下项目运行和验证所需文件：

- `src/`
- `package.json`
- `package-lock.json`
- `vite.config.*`
- `tsconfig*.json`
- `docs/V1.0-定版说明.md`
- `docs/V1.0-发布确认报告.md`
- `docs/V1.0-需求覆盖矩阵.md`
- `README.md`，如存在。

说明：

- 发布包应以空库形态交付。
- 发布包不应夹带真实试运行数据、验收库数据或真实备份文件。
- 若后续需要提供演示数据，应单独制作脱敏样例，不复用真实验收资料。

## 2. 内部保留，不建议外发

以下资料用于内部验收、审计和问题追溯，不建议随普通发布包外发：

- `release-evidence/v1.0/`
- `data/trial/`
- 真实 JSON 备份。
- 真实 Excel 报表。
- 验收库备份。
- 验收环境日志。
- 真实数据隔离试运行输入和输出。

原因：

- 这些资料可能包含真实或近真实业务数据。
- 这些资料用于证明 V1.0 验收闭环，不属于普通使用者运行系统所必需的文件。
- 外发前如确需提供，应先做脱敏、压缩范围确认和版本留档。

## 3. 不应包含

正式发布包不应包含以下文件或目录：

- `.env`
- 密钥文件。
- API Key、Token、私钥、密码或其他敏感配置。
- `node_modules/`
- 临时日志。
- 旧缓存。
- `dist/`，除非本次发布明确采用静态构建产物交付。
- 真实业务数据。
- 真实备份文件。
- 浏览器 Profile。
- 验收库本地数据。

## 4. 发布前检查

发布前建议再次确认：

- `npm.cmd run test` 通过。
- `npm.cmd run lint` 通过。
- `npm.cmd run build` 通过。
- `docs/V1.0-发布确认报告.md` 结论为可以正式定版 V1.0。
- 发布包中不包含 `.env`、密钥、真实数据、验收备份和浏览器 Profile。
- 发布对象明确知道当前版本是本地网页版，不是桌面离线版。

## 5. 建议发布形态

- 面向开发或内部验收：源码包。
- 面向普通使用者：优先提供空库构建产物或后续桌面安装包。
- 如需附带文档：仅附带 V1.0 定版说明、发布确认报告和必要使用说明。
